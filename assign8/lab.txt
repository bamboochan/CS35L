The part that is relevant to dynamic linking is
access("/etc/ld.so.preload", R_OK)      = -1 ENOENT (No such file or directory)
......
mprotect(0x7eff284bb000, 4096, PROT_READ) = 0

Specifically,open, fstat, mmap, read, close and mprotect are relevant to dynamical linking.
Dynamic Linking need to take the .so file, dynamically allocate memory for them
the dynamic linker need to use open syscall to open the libary file, 
and fstat give information on the file, such as size. mmap is used to incorporate 
the library into the process' address space. 
read is used then to read the data from the linker elf
All the above need mprotect to protect the memory
close closes the libraryafter all parts are dynamically linked
